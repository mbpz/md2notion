# 运行步骤

## 1. 安装依赖（首次或换环境时执行）

在项目根目录 `md2notion/` 下执行：

```bash
# 进入项目目录
cd /path/to/md2notion

# 创建虚拟环境（避免污染系统 Python）
python3 -m venv .venv

# 激活虚拟环境
source .venv/bin/activate   # macOS/Linux
# Windows:  .venv\Scripts\activate

# 安装依赖
pip install -r requirements.txt

#（可选）安装为可执行包，获得 notion-md-sync 命令
pip install -e .
```

## 2. 配置环境变量

在项目根目录创建 `.env` 文件（可复制 `.env.example` 再改），填入：

```bash
NOTION_TOKEN=secret_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
ROOT_PAGE_ID=xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
```

- `NOTION_TOKEN`：在 [Notion Integrations](https://www.notion.so/my-integrations) 创建 Integration 后获得。
- `ROOT_PAGE_ID`：Notion 里作为「根页面」的页面 ID（URL 中页面 ID 部分，支持带或不带连字符的 32 位 hex）。

并确保在 Notion 中已把该**根页面** Share 给你的 Integration。

## 3. 执行同步脚本

在项目根目录、且已激活虚拟环境时执行：

```bash
# 加载 .env 并执行增强版同步（推荐）
source .venv/bin/activate
set -a && source .env 2>/dev/null; set +a
notion-md-sync ./markdown
```

或一条命令（在项目根目录下）：

```bash
source .venv/bin/activate && (set -a; source .env 2>/dev/null; set +a) && notion-md-sync ./markdown
```

- `./markdown` 可换成任意本地 Markdown 根目录，例如 `./knowledge`。
- 未安装为包时，可用：`python notion-md-sync ./markdown` 或 `python -m notion_md_sync.cli ./markdown`。

## 4. 常用参数

```bash
# 强制全量重新上传（忽略 .notionsync 中的 hash）
notion-md-sync ./markdown --force

# 提高并发数（默认 8）
notion-md-sync ./markdown --workers 16

# 显式指定 token 和 page-id（可不依赖 .env）
notion-md-sync ./markdown --token "$NOTION_TOKEN" --page-id "$ROOT_PAGE_ID"
```

## 5. 使用基础版脚本（可选）

若只用基础版、不做增量与并发：

```bash
source .venv/bin/activate
export NOTION_TOKEN="your_token"
export ROOT_PAGE_ID="your_root_page_id"
export MARKDOWN_DIR="./markdown"   # 可选，默认 ./markdown
python import_md_to_notion.py
```

---

**小结：日常使用最少三步**

1. `cd md2notion && source .venv/bin/activate`
2. `set -a && source .env && set +a`
3. `notion-md-sync ./markdown`
